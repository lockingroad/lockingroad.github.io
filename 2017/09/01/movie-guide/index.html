MVP下的Dagger2的图结构

Posted by Demonl on 2017-09-01

MovieGuide-master
公共接口是一个电影api，需要申请一个key。

tmdb_api_key=716e9b55af1d1585604f48a78d9d44ac

结构

MVP

大致分为3部分

  • MoviesListingFragment显示电影列表
  • SortingDialogFragment菜单操作
  • MovieDetailsFragment详情

MoviesListingFragment
点击的动作在MoviesListingActivity中回调进行;
presenter和view的关联：moviesPresenter.setView(this);

SortingDialogFragment
采用不同的排序策略;

除了自己的presenter还共用的一个presenter。

MovieDetailsFragment
点击喜欢的电影

dagger2


点击
这是dagger2形成的图结构。

  1. 矩形框Component
  2. 椭圆框Module
  3. 横线输出
  4. 菱形输入

MoviesListingFragment和SortingDialogFragment
注入的时候用的是ListingComponent，放入这个模块比较恰当，Component的分类应该体现一定的结构差异，以便管理维护，但是不宜分的太细。

可以借鉴的地方

  • [x] 项目中的model模块是Interactor，操作网络和本地数据。本地化的数据在Store中进行了封装，如FavoritesStore、SortingOptionStore。
