基本知识

Request格式

  1. 请求行：请求类型 资源 协议版本
  2. 请求头部 （HOST请求目的地 User-Agent）
  3. 空行
  4. 请求数据：post的内容

客户端请求的时候，可以使用Accept字段声明自己可以接受哪些数据格式。

GET请求

GET /books/?sex=man&name=Professional HTTP/1.1
Host: www.wrox.com
User-Agent: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.7.6)
Gecko/20050225 Firefox/1.0.1
Connection: Keep-Alive

最后一个空行
get请求数据附在URL中 ？分割URL和传输数据，多个参数用&连接

POST请求

POST / HTTP/1.1
Host: www.wrox.com
User-Agent: Mozilla/5.0 (Windows; U; Windows NT 5.1; en-US; rv:1.7.6)
Gecko/20050225 Firefox/1.0.1
Content-Type: application/x-www-form-urlencoded
Content-Length: 40
Connection: Keep-Alive
空行
name=Professional%20Ajax&publisher=Wiley

POST提交的数据放置在http包的包体中。POST的请求参数在http标题的不同部分传输（entity body）传输的，传输表单信息，必须添加application/x-www-form- urlencoded。

Response格式
  1. 状态行：又http协议版本号 状态码 状态消息
  2. 消息报头：
  3. 空行
  4. 响应正文

MIME type在尾部使用分号，添加参数。

Content-Type: text/html; charset=utf-8

使用

结构

call执行时候的核心代码：

1.    在配置 OkHttpClient 时设置的 interceptors；
2.    负责失败重试以及重定向的 RetryAndFollowUpInterceptor；
3.    负责把用户构造的请求转换为发送到服务器的请求、把服务器返回的响应转换为用户友好的响应的 BridgeInterceptor；
4.    负责读取缓存直接返回、更新缓存的 CacheInterceptor；
5.    负责和服务器建立连接的 ConnectInterceptor；
6.    配置 OkHttpClient 时设置的 networkInterceptors；
7.    负责向服务器发送请求数据、从服务器读取响应数据的 CallServerInterceptor。

这是责任链模式，高级包装功能在上层，基础核心功能在下层
okhttp3.internal.connection.ConnectInterceptor

HttpCodec中利用okio对socket的读写操作进行封装

okhttp3.internal.http.CallServerInterceptor
